# Tests Matrix

Below is a consolidated overview of the tests from `tests/extractor` and `tests/chunker`. Threshold values are listed using the default values from the CLI arguments of the scripts themselves.

## Extractor

| Test Name | What It Tests | Brief Explanation of Terms / Approach / Metrics | What It Prints | Warning Thresholds | Fail Thresholds |
| --- | --- | --- | --- | --- | --- |
| `sanity.py` | Basic sanity checks for `out/pages.jsonl`: page sequence, correctness of `raw_text` / `clean_text`, empty or too-small pages, unexpected empty pages, `clean_text` expansion relative to `raw_text`, duplicate pages by hash. | `core pages` = pages without special labels (`toc-like`, `figure/table-heavy`, `separator-like`); `tiny` = `0 < len(clean_text) <= 120`; `unexpected empty` = an empty page between neighbors with text `>= 300`; `length violation` = `clean_text` after whitespace removal is longer than `raw_text` by more than `1.10x`; `duplicate_hash_extra` = extra pages in groups with identical SHA1 of normalized `clean_text`. | A page summary, label summary, sample page lists, `WARNINGS` and `FAILS` blocks, and final `OK/WARN/FAIL` status. | `empty_total_warn >= 5%`; `tiny_total_warn` = disabled by default (`None`); `empty_core_warn >= 5%`; `tiny_core_warn >= 8%`; `unexpected_empty_core_warn >= 5%`; `len_core_warn >= 10%`; `len_total_warn` = disabled; `len_ws_only_total_warn >= 50%`; `duplicate_warn >= 1%`. | `pages != 1..N`; `expected_pages_count` mismatch; invalid fields; `unexpected_empty_core_fail >= 10%`; `len_core_fail >= 50%`; `len_total_fail` = disabled; `duplicate_fail >= 3%`. |
| `cleanup_quality.py` | Garbage leakage after cleanup: chapter headers, URLs, inline footnote refs, math glyphs, merged long words. | `header_leak` = `clean_text` starts with a header from metadata; `url_leak` = `http/https/www` remains in the text; `inline_footnote_ref` = refs like `[12]`; `math_glyph_leak` = symbols from Mathematical Alphanumeric Symbols; `long_word_merge_leak` = a suspiciously long word that can be segmented into dictionary-like parts. | `pages=<N>`, then hit counts for each category, sample pages/tokens, and with `--verbose` also sample matches; then final status. | No separate warning thresholds. | Any violation (`total_violations > 0`) gives `FAIL` unless `--report-only` is enabled. |
| `rules_metrics.py` | Metrics for enabled cleanup-rule hits from `rules_metadata.json` over `clean_text` or `raw_text`. | This is not a gate test but a coverage/frequency counter. For line/block/text rules it counts matches and sample pages; with `--show-mode` it shows the matching mode (`line_rule`, `regex`, `anchor_regex`). | `pages=<N>`, `field=<...>`, then one line per rule in the form `<rule_name>: <matches> samples=[...]`. | None. | Only structural input errors: missing/invalid input, invalid `rules_metadata`, empty JSONL, invalid `page` field. There are no thresholds on the metrics themselves. |
| `synthetic_regression.py` | Regression-style synthetic cases for `clean_page_text`: whether cleanup on controlled inputs produces the expected `clean_text`. | Each case defines either `raw_text` or a set of `words` / page geometry; comparison uses normalized text. Useful for edge cases around Figure captions, line gaps, heading inference, and cleanup rules. | For each case `OK [name]` or `FAIL [name]`; on failure prints `expected`, `actual`, and optionally a token diff; at the end `summary: total=... failed=... passed=...`. | None. | Any case where `normalize(actual) != normalize(expected)` yields `FAIL`; also fail on invalid cases / metadata / rules format. |

## Chunker

### Common

| Test Name | What It Tests | Brief Explanation of Terms / Approach / Metrics | What It Prints | Warning Thresholds | Fail Thresholds |
| --- | --- | --- | --- | --- | --- |
| `truth_consistency.py` | How well the chunk corpus text covers the source `clean_text` across the full corpus. | `plain tokens` = words matched by regex `[A-Za-z0-9']+`; `tech tokens` = tokens with `_ : - . /` to catch degradation of technical notation; `recall` = share of source tokens found in chunks; `precision` = share of chunk tokens confirmed by source; `char_ratio` = `len(chunks_norm)/len(source_norm)`; `dup_factor` = extra token mass in chunks relative to source. | `corpus`, `plain`, `tech` blocks, then `WARNINGS` and `FAILS` tables, and final `OK/WARN/FAIL`. | `plain_recall < 0.955`; `plain_precision < 0.994`; `plain_dup_factor > 0.01`; `tech_recall < 0.93`; `tech_precision < 0.97`; `tech_dup_factor > 0.025`; `char_ratio < 0.93` or `> 1.02`. | Empty inputs / empty scope; `plain_recall < 0.94`; `plain_precision < 0.992`; `plain_dup_factor > 0.02`; `tech_recall < 0.90`; `tech_precision < 0.955`; `tech_dup_factor > 0.045`; `char_ratio < 0.90` or `> 1.03`. |
| `sanitation.py` | Sanity heuristics over chunk artifacts: size, repetitions, cuts in the middle of words, strange `section_path` tails. | `tiny/huge` = chunk shorter than `250` or longer than `12000` characters; `repeated_sentence_ratio` = share of chunks with repeated sentence-like fragments; `midword_ratio` = heuristic for a broken word / hyphen split at the chunk boundary; `chunk_name_mismatch_ratio` = `section_path` tail does not resemble the title expected for `page_start` from metadata. | Length distribution (`min/p50/p90/p95/p99/max`), tiny/huge counts, a `violations` block, then `warnings` and `fails` tables, and final status. | `tiny_ratio > 0.10`; `huge_ratio > 0.02`; `repeated_sentence_ratio > 0.04`; `midword_ratio > 0.05`; `chunk_name_mismatch_ratio > 0.01`. | Empty input; `tiny_ratio > 0.20`; `huge_ratio > 0.05`; `repeated_sentence_ratio > 0.08`; `midword_ratio > 0.10`; `chunk_name_mismatch_ratio > 0.02`. |
| `determinism.py` | Chunker determinism on the same input across two consecutive runs. | It compares the `stable_projection` over fields `chunk_index`, `page_start`, `page_end`, `section_path`, `text`; volatile fields such as `chunk_id` / `chunk_created_at` are ignored. | `chunks=<N> mismatches=<K>`, optionally `field_mismatch_counts={...}`, then example differences by line and final status. | No separate warning thresholds; only `--report-only`. | Any mismatch in any line/field between run1 and run2 yields `FAIL`; also fail if files are missing or chunk counts differ. |
| `schema_validation.py` | Contract validation: every record in `chunks.jsonl` must conform to `chunk_schema.json`. | Checks required fields, `additionalProperties=false`, types, `date-time` fields, array-valued `section_path`, and the business rule `page_end >= page_start`. | `chunks=<N> invalid=<K>`, `violation_counts={...}`, optionally sample mismatches, and final status. | No separate warning thresholds; only `--report-only`. | Any invalid record, empty input, or page-span contract violation yields `FAIL`. |

### Structural

| Test Name | What It Tests | Brief Explanation of Terms / Approach / Metrics | What It Prints | Warning Thresholds | Fail Thresholds |
| --- | --- | --- | --- | --- | --- |
| `span_quality.py` | Positional quality of structural chunks: whether each chunk can be mapped back to source, and whether order, coverage, and page-span metadata are preserved. | `locate_span` looks for an exact or anchor match; `coverage_effective` = source coverage after subtracting meaningful holes; `hole` = a significant text gap between chunks; `overlap` = overlap of adjacent mapped spans; `page violation` = `chunk.page_start/page_end` poorly aligned with inferred page start/end. For the structural chunker it also matters that chunk boundaries align well with metadata-driven section spans. | `corpus`, number of matched chunks, matching methods (`exact`, `anchor`, `*_unstripped`), counts by violation type, `coverage_raw`, `coverage_effective`, warnings/fails blocks, and final status. | `coverage_effective < 0.99`; `boundary_warning_ratio > 0.10`. | Empty inputs; `chunk_index` not `0..N-1`; any `miss`, `order_viol`, `overlap_viol`, `hole_viol`, `page_viol`; `coverage_effective < 0.98`. With `overlap_mode=off`, also fail if overlap `> 40` characters. |
| `structure.py` | Structural chunks vs metadata structure: heading-id order, no missing/duplicate/unexpected ids, valid `section_path`, and title consistency in the chunk heading. | `chunk heading` = chunk start `"<id> <title>"`; `jump violation` = sibling-id jump greater than 1; `path_hierarchy_violations` = the path is missing from metadata or a parent path appears “in the middle” of its descendants; `title_mismatch_ratio` = share of chunk headings whose title matches the metadata title poorly within the first `220` characters. | Summary of `chunks`, `scope_page_span`, number of expected/found headings, metadata counts, violation counts with samples, a line about `title_consistency_checked`, and final status. | No separate warning thresholds. | Empty inputs; `chunk_index` not `0..N-1`; any `order_viol`, `jump_viol`, `duplicate_heading_ids`, `missing_heading_ids`, `unexpected_heading_ids`, `path_hierarchy_violations`; `title_mismatch_ratio > 0.20`. |
| `synthetic_regression.py` | Controlled synthetic regression cases for the structural chunker: correct segmentation by metadata structure, section paths, chunk count, and absence of forbidden paths. | Each case writes temporary `pages.jsonl` and `metadata.json`, launches the structural chunker, then checks `expected_paths`, `expected_chunk_count`, `forbidden_paths`, and anchor-based checks inside the required `section_path`. | For each case `[OK]` or `[FAIL]` with an error list; at the end `cases=<N> failed=<K>` and overall status. | No separate warning thresholds; only `--report-only`, where regressions become non-blocking. | Any mismatch in path sequence, count, forbidden path, or anchor check fails the case; if there is at least one failure and no `--report-only`, the run ends with `FAIL`. |

### Fixed

| Test Name | What It Tests | Brief Explanation of Terms / Approach / Metrics | What It Prints | Warning Thresholds | Fail Thresholds |
| --- | --- | --- | --- | --- | --- |
| `sentence_stream.py` | Correctness of the intermediate `SentenceUnit` stream for the fixed chunker. | Checks marker-aware sentence segmentation, marker removal, `page_start/page_end` on the `SentenceUnit` level, absence of empty sentence units, and correct `token_count`. This is a stage-level test for the fixed pipeline `pages -> doc_stream -> sentence_stream`. | `sentences=<N> invalid=<K>`, counts by violation type (`marker_leak`, `empty_sentence`, `page_range_error`, `token_count_error`), sample cases, and final status. | No separate warning thresholds; only `--report-only`. | Any marker leak, empty `SentenceUnit`, invalid `page_start/page_end`, sentence-stream order violation, or absence of sentence units yields `FAIL`. |
| `page_mapping.py` | Page coverage quality in the fixed chunker after marker-aware segmentation and at the final chunk level. | `sentence page coverage` = page range of each `SentenceUnit`; `chunk page coverage` = aggregated page range of a chunk; checks duplicate-page handling, empty-page preservation, cross-page sentence spans, monotonic page ranges, and no loss of page coverage. | `sentences=<N> chunks=<M>`, counts by mapping violation type, sample spans/chunks with expected vs actual page coverage, and final status. | No separate warning thresholds; only `--report-only`. | Any invalid `SentenceUnit.page_start/page_end`, any invalid `chunk.page_start/page_end`, loss of an empty page inside the coverage range, or violation of stable duplicate-page ordering yields `FAIL`. |
| `overlap.py` | Correctness of overlap semantics for the fixed chunker. | Checks that adjacent chunks overlap by suffix sentences, that overlap is computed by token budget, that it does not cut sentences, that it respects `overlap_ratio=0`, and that it does not prevent progress. | `chunks=<N> overlap_checks=<K> failed=<M>`, sample adjacent chunk pairs, and final status. | No separate warning thresholds; only `--report-only`. | Any violation of the overlap contract, lack of progress, sentence splitting, or incorrect behavior at `overlap_ratio=0` yields `FAIL`. |
| `synthetic_regression.py` | Controlled synthetic regression cases for the fixed chunker: sentence-based packing, overshoot, long-sentence rule, duplicate pages, empty lines, empty pages, page-marker handling, section annotation fallback/error cases. | Each case writes temporary `pages.jsonl`, `book_metadata.json`, `content_metadata.json`, `chunker.toml`, launches the fixed chunker, and checks expected chunk count, expected texts, expected page ranges, expected section paths, and expected hard failures. | For each case `[OK]` or `[FAIL]` with an error list; at the end `cases=<N> failed=<K>` and overall status. | No separate warning thresholds; only `--report-only`, where regressions become non-blocking. | Any mismatch in expected chunks/text/page ranges/section paths or expected failure semantics fails the case; if there is at least one failure and no `--report-only`, the run ends with `FAIL`. |
